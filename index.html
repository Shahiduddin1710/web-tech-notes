Hypertext Transfer Protocol (HTTP)

HTTP

HTTP is a stateless, application-level protocol used for communication between a web browser and a web server.

HTTP defines how clients and servers exchange data.

Methods: GET, POST

Includes headers, status codes, and payload.

Handles communication only, not security.

HTTPS (Secure)

HTTPS is the secure version of HTTP that uses SSL/TLS encryption.

It wraps HTTP inside a secure cryptographic tunnel.

Application logic, APIs, and routes remain unchanged.

Only the transport layer gains encryption, integrity, and authentication.

Technical Differences

1) Encryption

TLS handshake negotiates keys making packets unreadable.

2) Integrity

MAC/AEAD ensures data cannot be modified.

3) Server Identity

SSL certificates verify domain authenticity.

4) Cookie Security

Secure and HttpOnly cookies reduce session hijacking.

5) Modern Web Support

Required for HTTP/2, HTTP/3, Service Workers, PWAs.

6) Ports

HTTP uses Port 80

HTTPS uses Port 443

Differentiation

1) Definition

HTTP: HTTP (Hyper Text Transfer Protocol) is a stateless application layer protocol used for communication between a web browser and a web server.

HTTPS: HTTPS (Hyper Text Transfer Protocol Secure) is the secure version of HTTP that uses SSL/TLS encryption to protect data.

2) Security

HTTP: Does not provide security. Data is sent in plain text and can be intercepted.

HTTPS: Provides security by encrypting the data during transmission.

3) Encryption

HTTP: Does not use encryption.

HTTPS: Uses SSL/TLS encryption. During the TLS handshake, secret keys are generated and data becomes unreadable.

4) Integrity

HTTP: Data can be modified during transmission.

HTTPS: Ensures integrity using MAC/AEAD. If data is altered, the connection breaks.

5) Server Identity & Authentication

HTTP: Does not verify server identity.

HTTPS: Uses SSL certificates to verify domain identity and protect against MITM attacks.

6) Browser & Platform Support

HTTP: Marked as "Not Secure" and some APIs may be blocked.

HTTPS: Trusted by browsers and required for modern web features.

HTTP Status Codes

1XX – Informational Status Codes

Indicate that the request has been received and processing is continuing.

100 – Continue: The server has received the initial part of the request.

101 – Switching Protocols: The server switches protocols as requested by the client (for example, upgrading from HTTP to WebSockets).

2XX – Successful Status Codes

Indicate that the request was successfully received, understood, and processed.

200 – OK: The request was successful, and the server returned the requested data.

201 – Created: A new resource has been successfully created (commonly after form submission or data creation).

204 – No Content: The request was successful, but the server has no content to return.

3XX – Redirection Status Codes

Indicate that the requested resource is located elsewhere and further action is required to complete the request.

301 – Moved Permanently: The resource has permanently moved to a new URL. Search engines update their records accordingly.

302 – Found (Temporary Redirect): The resource is temporarily available at a different location.

304 – Not Modified: The requested resource has not changed since the last request, so the browser can use its cached version.

4XX – Client Error Status Codes

Indicate that there is a problem with the client-side request.

400 – Bad Request: The server cannot understand the request due to incorrect syntax or invalid data.

401 – Unauthorized: Authentication is required, and the client has not provided valid credentials.

403 – Forbidden: The client is authenticated but does not have permission to access the resource.

404 – Not Found: The requested resource does not exist on the server.

5XX – Server Error Status Codes

Indicate that the server failed to fulfill a valid request.

500 – Internal Server Error: A generic error indicating that the server encountered an unexpected condition.

502 – Bad Gateway: The server received an invalid response from an upstream server.

503 – Service Unavailable: The server is unable to handle the request due to overload or maintenance.

504 – Gateway Timeout: The server did not receive a timely response from another server.

Version Control System

A Version Control System (VCS) is a tool designed to track changes in file history and enable multiple developers to work on the same project efficiently.

Git is a distributed version control system, while GitHub is a cloud-based platform that hosts Git repositories and enables collaboration.

Git

Git is an open-source distributed version control system created by Linus Torvalds.

It allows developers to track changes in source code, revert to previous versions, create branches for experimentation, and collaborate efficiently.

Key Characteristics

  • Distributed system
  • High performance
  • Offline capability
  • Data integrity

Git Commands

git init: Initializes a new repository.
git status: Displays the current state of the working repository and shows modified files.
git add: Adds files to the staging area and prepares them for committing.
git commit: Saves staged changes permanently in the repository. Each commit has a unique hash ID, author information, timestamp, and commit message.
git log: Displays the commit history and is useful for tracking project evolution.
git branch: Used to create and manage branches.
git checkout: Switches between branches.
git merge: Combines changes from one branch into another.

CSS Dimensions

1) Absolute Units

px

A fixed unit based on screen pixels. It does not change relative to other elements.

cm

A size measured in centimeters. Mostly used for print-based layouts.

mm

A size measured in millimeters. Also mainly used for print-based layouts.

2) Relative Units

%

Relative to the size of the parent element.

Example: width: 50%;

em

Relative to the font size of the parent element.

Example: font-size: 2em;

rem

Relative to the root (html) font size.

Example: font-size: 1.5rem;

vw

1% of the viewport width.

Example: width: 50vw;

vh

1% of the viewport height.

Example: height: 100vh;

CSS Position

Static

The default positioning. Elements follow the normal document flow.

Relative

Moves relative to its original position.

position: relative; top: 10px;

Absolute

Positioned relative to the nearest positioned (non-static) parent element.

position: absolute; top: 0;

Fixed

Stays fixed relative to the viewport, even when scrolling.

position: fixed; bottom: 0;

Sticky

A hybrid of relative and fixed positioning. It behaves as relative until a defined scroll position is reached.

position: sticky; top: 0;

Floating

Originally used for layouts. It allows elements to float left or right.

float: left;

PART I – JavaScript

JavaScript is an interpreted, high-level programming language.

It is case-sensitive.

Statements typically end with a semicolon (;).

Code blocks use curly braces { }.

Variables

Variables are containers used to store values.

Three types of variables:

var

Function-scoped

const

Block-scoped (cannot be reassigned)

let

Block-scoped (can be reassigned)

Difference Between var and let

1) Scope Difference

var: Function-scoped. It is accessible anywhere inside the function.

let: Block-scoped. It is accessible only within the block in which it is declared.

2) Redeclaration

var: Redeclaration is allowed.

let: Redeclaration is not allowed in the same scope.

Hoisting

var:
console.log(a); // undefined
var a = 5;

let:
console.log(b); // ReferenceError (Temporal Dead Zone)
let b = 10;

Global Object Behavior

var:
var a = 5;
console.log(window.a); // 5

let:
let a = 8;
console.log(window.a); // undefined

PART II – Document Object Model (DOM)

The Document Object Model (DOM) is a programming interface that allows JavaScript to interact with HTML and CSS.

When a webpage loads, the browser parses the HTML document and converts it into a tree-like structure known as the DOM Tree.

In this structure, every HTML element becomes an object (node), every attribute becomes a property, and every piece of text becomes a text node.

JavaScript can use the DOM to dynamically modify content, structure, and styling after the page has loaded.

The DOM makes webpages interactive instead of static by allowing real-time updates without reloading the page.

It acts as a bridge between HTML (structure), CSS (styling), and JavaScript (behavior).

The DOM is maintained as a living standard by WHATWG and continuously evolves with browser improvements.

DOM Tree Structure

Nodes in DOM

1) Document Node – Represents the entire HTML document.

2) Element Node – Represents HTML elements like div, p, h1.

3) Text Node – Represents the text inside elements.

4) Attribute Node – Represents element attributes such as id and class.

DOM Traversal

DOM traversal means navigating between elements in the DOM tree.

Important properties include parentNode, children, nextElementSibling, and previousElementSibling.

This allows developers to move between related elements efficiently.

DOM Manipulation

DOM manipulation refers to adding, removing, or modifying elements dynamically.

Common methods include createElement(), appendChild(), remove(), and setAttribute().

Developers can also modify text using textContent or HTML using innerHTML.

Event Handling in DOM

Event Listeners

Events are actions detected by the browser such as click, keydown, submit, and mouseover.

JavaScript uses addEventListener() to respond to user interactions.

This enables interactive features like buttons, forms, and animations.

Event Bubbling

Event bubbling occurs when an event starts at the target element and propagates upward to parent elements.

This behavior allows parent elements to handle events from their children.

Event Delegation

Event delegation is a technique where a parent element handles events for multiple child elements.

It improves performance and works for dynamically added elements.

DOM Levels

1) DOM Level 0

Unofficial and browser-dependent implementation.

Included basic objects like window and document.

2) DOM Level 1

The first official standard focusing on document structure and basic manipulation.

3) DOM Level 2

Introduced event handling and CSS manipulation.

Made webpages more dynamic and interactive.
